# scottxu.dev

Personal portfolio for **Scott Xu** — an AI Product Builder with deep banking-domain
expertise who also ships his own products. Built with [Astro](https://astro.build)
as a static site, mobile-first, bilingual (English default + Chinese mirror), and with
a first-class **machine-readable layer** for AI recruiting agents and LLM crawlers.

## Run it

```bash
npm install
npm run dev      # local dev server → http://localhost:4321
npm run build    # static build → ./dist
npm run preview  # serve the built ./dist locally
```

Requires Node ≥ 22.12 (Astro 7 requirement; built and tested on Node 24).

### The single email config value

The email lives **once** in `src/data/site.ts`:

```ts
email: 'xu.zhihao@u.nus.edu',   // → switch to 'hello@scottxu.dev' when ready
```

Changing it there updates the hero CTA, the footer, and the Person JSON-LD.

> ⚠️ **Two crawler-facing copies are static files** and are NOT auto-generated
> from `site.ts`. If you change the email, also update:

## Chinese (中文) content

- English is the default site at `/`. The Chinese mirror lives at `/zh/`.
- **URLs are never translated** — the same English slugs are used on both sides,
  cross-linked via `hreflang`.
- The Chinese copy is **not** machine-translated. Every ZH string is a
  `TODO(zh)` placeholder for you to write by hand. Find them all with:

```bash
# from the project root
grep -rn "TODO(zh)" src
```

They live in `src/i18n/ui.ts`, `src/data/experience.ts`, `src/data/skills.ts`,
and `src/data/projects.ts`.

## Machine-readable layer (the differentiator)

| Resource | Where it comes from |
| -------- | ------------------- |
| **Person JSON-LD** (home) | `src/data/jsonld.ts` → `personSchema()`, injected by `BaseLayout` |
| **CreativeWork JSON-LD** (each project page) | `src/data/jsonld.ts` → `creativeWorkSchema()` |
| **`/llms.txt`** | `public/llms.txt` (plain-Markdown self-summary for LLMs) |
| **`/resume.json`** | `public/resume.json` ([JSON Resume](https://jsonresume.org) schema) |
| **`/sitemap.xml`** | generated by `src/pages/sitemap.xml.ts` (with hreflang alternates) |
| **`/robots.txt`** | `public/robots.txt` (allows all crawlers, points to sitemap) |

After a build you can sanity-check them:

```bash
npm run build
ls dist                       # index.html, zh/, projects/, sitemap.xml, llms.txt, resume.json, robots.txt
```

## Deploy to Cloudflare Pages

This is a plain static build — no adapter needed. **Nothing here is deployed yet;**
do this when you're ready.

### Option A — Git integration (recommended)

1. Push this repo to GitHub/GitLab.
2. Cloudflare dashboard → **Workers & Pages → Create → Pages → Connect to Git**.
3. Build settings:
   - **Framework preset:** Astro
   - **Build command:** `npm run build`
   - **Build output directory:** `dist`
4. Deploy, then add the custom domain **scottxu.dev** under the project's
   **Custom domains** tab (Cloudflare will provision the DNS + TLS).

### Option B — Direct upload with Wrangler

```bash
npm run build
npx wrangler pages deploy dist --project-name scottxu-site
```

### Notes

- `public/_headers` sets security headers and correct content types for
  `llms.txt` / `resume.json`; Cloudflare Pages applies it automatically.
- The site origin is set in `astro.config.mjs` (`site: 'https://scottxu.dev'`).
  If you deploy to a different domain, update it there so canonical/hreflang/sitemap
  URLs stay correct.
